## v2.0 — Near-Term Enhancements

**Current:** 2 vehicles validated; up to 4 in simulation  
**Target:** 8–16 vehicles approaching simultaneously from all directions

- Extend FCFS to a multi-lane priority queue
- Add staggered-crossing support (2 non-conflicting vehicles crossing at once)
- Handle mixed traffic: autonomous + human-driven vehicles in same queue

### 4. IEEE 802.11p / DSRC Hardware

**Current:** Standard Wi-Fi 802.11ac simulated as V2V channel  
**Target:** Dedicated DSRC radio (5.9 GHz) with sub-10 ms latency

- Replace `CommunicationChannel` with real `DSRC_Channel` backed by socket/UDP
- Implement WAVE (Wireless Access in Vehicular Environments) message set
- Test in actual outdoor vehicle-to-vehicle range experiments

### 5. Dynamic Traffic Adaptation (Reinforcement Learning)

Replace static FCFS with a learned policy:

```
State:  [vehicle positions, velocities, queue length, time of day, weather]
Action: [assign priority to vehicle X]
Reward: +1 for safe crossing, -10 for collision, -0.1 per second of wait time
```

Framework: Stable-Baselines3 (PPO / SAC) running on RPi 5 with ONNX export

### 6. Smart City Integration

- REST API endpoint on RSU exposing live intersection telemetry
- MQTT broker (Mosquitto) for city-wide traffic dashboard integration
- Integration with OpenStreetMap + SUMO for real road topology

### 7. Fault Tolerance

**Current:** No retry or fallback if V2I link drops  
**Target:**

- V2I heartbeat monitoring; fall back to V2V-only FCFS if RSU unreachable
- Message sequence numbers + selective retransmission
- Vehicle-side watchdog: if no priority received in 2 s, enter safe-hold mode

### 8. Edge AI on Raspberry Pi 5

Deploy TensorFlow Lite / ONNX model directly on the RPi NPU (if model < 50 MB):

```bash
# Convert trained LSTM to TFLite
tflite_convert \
  --saved_model_dir=model/lstm_predictor \
  --output_file=model/lstm_predictor.tflite
```

No cloud dependency — all inference on-device, sub-20 ms latency.

### 9. LiDAR / Camera Sensor Fusion

| Sensor | Adds |

### 10. Multi-Intersection Coordination

Federated RSU network where neighboring RSUs share queue state over a backbone link, enabling:

- Green-wave optimization across sequential intersections
- Emergency vehicle corridor clearing across the city grid
